Reboot CSS

La première partie du CSS cde bootstrap est un "reboot", c'est un dire toute une partie CSS qui permet de remettre en mode web et accessible les éléments normalement couvert par le CSS du navigateur, qui n'est pas accessible par défaut.

La police de caratère est modifiée, passant d'une police à empatement non adaptée à l'affichage à l'écran par une police sans empatement. La taille de la police par défaut est augmentée ainsi que l'interligne.

Tous les éléments sont par défaut mis en calcul de taille avec les bordures, alors que le CSS navigateur gère les tailles des éléments par leur contenu. De plus, la marge extérieure de body est supprimée.

Premièrement, il gère les titres, puis les paragraphes sont modifiés.

Les liens sont modifiés.

Le séparateur horizontal est <hr /> modifié

  • Les liste comprises à l'intérieur d'élément de liste n'ont pas de marge extérieure basse

Les media <img />

Pour rendre les media adaptatifs, Bootstrap propose des classes adaptées.

Pour les images, la classe "img-fluid" permet d'afficher l'image jusqu'à 100% de sa largeur, mais elle ne s'affichera jamais plus grande que sa taille originale.

Les media <picture>

la classe img-fluid peut être aussi utilisée avec la balise <picture>

Les media : vignettes

on peux créer les images sous forme de vignette avec img-thumnail et faire "flotter" les images avec float-start sur la gauche et float-end sur la droite

On peut center l'image à l'aide des classes mx-auto et d-block

Les Media <video>

Quand les vidéos sont hébergées sur le site, on utilise la balise <video>, cette balise peut utiliser la classe "img-fluid" pour la rendre directement responsive. Cette balise permet aussi de proposer à l'utilisateur plusieurs format de vidéo, comme ceci si son navigaeur n'accepte pas ou ne peux pas lire un format, il peux tenter de lire un autre format.

Si on crée un site illustré par beaucoup de vidéo, il est intéressant d'héberger ses vidéo sur un site spécalisé, comme youtube, et de profiter du partage via iframe qu'il propose.

En revanche, le partage par défaut des vidéos sur youtbe n'est pas parfait, il propose une iframe avec une largeur et une hauteur fixe, non compatible avec la navigation mobile.

Bootstrap propose des classes qui permettent de rendre les iframe responsive, on intègre l'iframe dans un conteneur qui aura deux classe, celle qui prépare le conteneur et sont contenu à afficher une iframe responsive et une classe qui indique le ratio de la vidéo (19/9, 21/9, 4/3, etc.)

Les media <audio>

La balise <audio> se comporte de la même façon que la balise <video>.

La balise <audio> affiche le lecteur du navigateur, il n'y a pas de contrôle css possible pour changer son apparence, idem avec la balise <video>. pour avoir un lecteur customisé, il faut le construire soit-même avec html, CSS et gérer les controls à l'aide du javascript.
